C++默认构造函数

Created2020-11-15|Updated2020-11-18
|Post View:

编译器只有在以下四种情况才会创建默认构造函数：

1.带有默认构造函数的类成员对象

2.带有默认构造函数的基类

3.带有一个虚函数的类

4.带有一个虚基类的类

以下为详细讲解

我在C++Primer7.1.4一节中曾看到说：

如果我们的类没有显式地定义构造函数，那么编译器就会为我们隐式地定义一个默认构造函数（ps：我曾以为，对于所有的没有显式定义构造函数的类，编译器都会为我们隐式地定义一个默认构造函数，但实际上并不是这样的）。

在Effective C++中条款5曾写到：惟有当默认构造函数被需要（被调用），它才会被编译器创建出来。

惟有当这些函数被需要（被调用），它们才会被编译器创建出来。那么很明显，我们这段代码，编译器并不需要默认构造函数，所以默认构造函数并不会被编译器创建出来。

那么，什么时候编译器需要默认构造函数并合成出一个默认构造函数呢？此外，被合成出来的默认构造函数只执行编译器所需的行动。也就是说，即使有需要为class Foo合成一个默认构造函数，默认构造函数也不会将两个数据成员初始化为0。为了让上一段代码正确执行，你必须提供一个显式的默认构造函数，将两个数据成员适当地初始化。

上面说过当编译器需要默认构造函数的时候，编译器才会合成它。下面4小节分别讨论什么时候编译器需要它。

1.“带有 默认构造函数”的类成员对象

如果一个class没有任何构造函数，但它内含一个类成员对象，而后者有默认构造函数，那么这个class的隐式地默认构造函数就是有用的，编译器需要为该 class 合成出一个默认构造函数。不过这个合成操作只有在构造函数真正需要被调用时才会发生。

被合成的Bar的默认构造函数内含必要的代码，能够调用class Foo的默认构造函数来处理成员对象Bar::foo，但它并不产生任何代码来初始化Bar::b。是的，将Bar::foo初始化是编译器的责任，将Bar::b初始化则是程序员的责任。被合成的默认构造函数看起来可能像这样：

编译器的行动是：“如果class A内含一个或一个以上的成员类对象，那么 class A 的每一个构造函数必须调用每一个类对象的默认构造函数”。编译器会扩张已存在的构造函数，在其中安插一些代码，使得在你的代码被执行之前，先调用必要的默认构造函数。延续前一个例子，扩张后的构造函数可能像这样：

2.带有默认构造函数的基类

类似的道理，如果一个没有任何构造函数的类派生自一个带有默认构造函数的基类，那么派生类的默认构造函数因被编译器需要所以被合成出来。它将调用上一层基类的默认构造函数（根据它们的声明顺序）。对一个后继派生的类而言，这个合成的默认构造函数和一个被显式提供的默认构造函数没有什么差异。

如果设计者提供多个构造函数，但其中都没有默认构造函数，和上一条一样，编译器会扩张现有的每一个构造函数，将合成默认构造函数的内容插入到每一个构造函数中去。

1.class声明（或继承）一个虚函数。

1.任何 class如果没有定义 default constructor，就会被合成出一个来。

如你所见，没有一个是真的！
